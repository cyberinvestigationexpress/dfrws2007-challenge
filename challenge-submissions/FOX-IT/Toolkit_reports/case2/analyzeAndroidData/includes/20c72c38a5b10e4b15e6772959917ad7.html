Q. How can I change the name of my Remote Desktop Session Host link in Remote Desktop Web Access to something other than the default name of Remote Desktop?
John Savill

A. With Remote Desktop Web Access, a list of published applications can be made available to users via a web interface. In addition a link, to create a desktop session on the Remote Desktop Session Host (formerly known as Terminal Server) can also be enabled, with a default name of Remote Desktop.

There might be instances when you want a more specific name, particularly if you have a Remote Desktop Web Access server that's displaying applications or desktops from multiple Remote Desktop servers—in that case, you'd have multiple Remote Desktop links on the desktop and no way of knowing which link pointed to which Remote Desktop Session Host.

Using the registry editor, you can change the name of the link on each Remote Desktop Session Host.

  1. Log on to the Remote Desktop Session Host server as an administrator.
  2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Terminal Server\TSAppAllowList\RemoteDesktops\TSRemoteDesktop
  3. Double-click Name and set it to the desired name (instead of Remote Desktop). Click OK.
  4. The new name will show in Remote Desktop Web Access after a while. You can also remove the Remote Desktop Session Host from Remote Desktop Web Access as a source then re-add it to make it show up straight away.

Another option is to use the WMI class documented on this Microsoft page to change the Name property.


Q. How can I hide the Remote Desktop tab of Remote Desktop Web Access?

A. Normally, a link to Remote Desktop is displayed along with the RemoteApp Programs in Remote Desktop Web Access, as shown here.



The link lets users manually type an RDP destination and desired screen size. If you don't want the Remote Desktop tab displayed, you can hide it by changing a setting in IIS Manager, , Sites, Default Web Site, RDWeb, Pages. Select Application Settings in the ASP.NET section and change the ShowDesktops value from true to false.



Once the change is made, the Remote Desktop tab will no longer be displayed.



Q. Am I allowed to use the Microsoft Desktop Optimization Pack (MDOP) Diagnostics and Recovery Toolkit (DaRT) on my servers?

A. MDOP is a per-device, per-year subscription that's available for computers covered by an enterprise agreement. It gives those devices access to all the parts of MDOP, including DaRT and its updated versions of many of the tools Microsoft acquired during the Winternals acquisition.

Because MDOP is licensed for desktops, does that mean you can't use it on your servers? If every desktop in your organization is covered by MDOP, you're allowed to use DaRT on your servers as well, but every desktop must have MDOP. There's no other way to get DaRT licensed for your servers.


Q. I'm trying to install various management tools on my Remote Desktop Session Host but I get failures related to contacting Active Directory. What's wrong?

A. Every time I've seen this, it's because IP virtualization is enabled on a per-session level. This causes some tools to get confused about how they should talk to the network. IP virtualization at a per-session level means each session gets its own IP address, rather than using the IP address of the Remote Desktop Session Host. Disabling IP virtualization solved the problem in each case.


Q. I've installed the Data Protection Manager (DPM) 2010 Management Shell on a computer but when I try to run the GUI I get an error saying the DPM service can't be found. Why?

A. The DPM 2010 Management Shell can be installed on non-DPM servers, letting PowerShell manage DPM remotely. As part of the DPM Management Shell installation, the DPM MMC snap-in console is included in the bin folder, but the DPM MMC console won't run remotely—it can only be run on the DPM server itself, which is why no link to the MSC file is placed on the Start menu during installation of the DPM 2010 Management Shell. If you want to run the DPM Management MMC snap-in, you need to log on (locally or remotely) to the DPM server.
